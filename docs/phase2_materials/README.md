# Phase 2 Materials

本目录将 `data/phase2` 中的散乱资料，连同 `data/phase1` 已有法规文本，整理成后端 Phase 2 可直接使用的标准材料。

## 目标

- 为 `template system` 提供可参数化模板
- 为 `POST /draft` 提供字段定义
- 为文书生成提供最小可引用的流程规则和法规依据

## 资料分类

### 1. 模板标准件

- `template_投诉信_商品质量.md`
- `template_投诉信_商品质量.json`
- `template_催告函_退款退货.md`
- `template_催告函_退款退货.json`
- `template_民事起诉状_网络购物纠纷.md`
- `template_民事起诉状_网络购物纠纷.json`

这些文件已经做成：

- 固定结构
- 占位字段
- 最小必填项
- 推荐引用法条

### 2. 流程规则

- `rule_12315投诉流程说明.md`
- `rule_网上立案流程说明.md`
- `rule_起诉材料要求.md`

这些文件的用途是：

- 指导 `/draft` 输出“下一步怎么做”
- 帮后端后续做“缺失材料提示”
- 帮前端或调用方理解结构化字段从哪里来

### 3. 法规摘录

- `law_民事诉讼法_起诉受理证据管辖摘录.md`
- `law_消费者权益保护法_退款退货平台责任摘录.md`
- `law_电子商务法_平台责任摘录.md`
- `rule_消费者权益保护法实施条例_投诉处理退货规则摘录.md`
- `rule_侵害消费者权益行为处罚办法_重点摘录.md`

这些文件不是完整法规，而是 Phase 2 真正会用到的摘录版，避免把整部法律都塞进模板生成链路。

### 可信度边界

- 法规摘录：来自用户整理的 `docx` 原文，适合做 RAG 依据和引用基础。
- 流程截图：主要是操作参考，适合总结字段、步骤和用户动作，不应单独视为权威法律依据。
- 模板标准件：是基于现有模板草稿、截图操作信息和法规摘录做的标准化整理，适合进入后端模板系统。

## 已完成的标准化决策

- 将“投诉信”聚焦为 `商品质量纠纷`
- 将“催告函”聚焦为 `退款/退货催告`
- 将“民事起诉状”聚焦为 `网络购物纠纷`
- 将截图信息转成文字规则，避免后续模板系统依赖图片
- 将 Phase 1 中已有法规整理为 Phase 2 摘录版，而不是重复造新资料

## 当前仍保留的缺口

- 大部分资料仍缺官方 `source_url`
- 12315 流程和网上立案流程当前主要依据用户截图，应在后续有条件时补充官方网页正文链接
- 还没有“已填写样例”文件
- 还没有模板入库清单或数据库 seed

## 后端下一步建议

1. 以三个模板 JSON 为基础，定义 `draft` 输入 schema。
2. 以三个模板 MD 为基础，定义模板渲染顺序和缺失字段提示。
3. 以流程规则文件为基础，给 `/draft` 增加 `next_steps`。
4. 以法规摘录为基础，给 `/draft` 增加 `suggested_citations`。
